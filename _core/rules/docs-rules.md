# docs-rules.md

이 프로젝트에서 작성하는 모든 마크다운 문서에 적용되는 필수 규칙이다.

## 1. 문체

- 본문은 서술체(`~이다`, `~한다`) 사용 - 경어체(`~입니다`, `~합니다`)는 문서 본문 금지
- 이모지 사용 금지
- 일반 한글/영문 키보드에서 직접 입력 가능한 문자를 사용
- em dash, curly quote 등 키보드로 직접 입력할 수 없는 문자 사용 금지 - 대신 `-`, `"`, `'` 사용
- 수학 및 기술 용어는 영어 단독 표기 (`batch normalization`, `learning rate`)
- 테이블 및 목록 바로 앞에 도입 문장 1문장 이상 삽입

## 2. 인코딩

- 마크다운 문서는 UTF-8 인코딩으로 생성하고 저장한다.
- CP949, EUC-KR 등 다른 인코딩으로 저장하지 않는다.
- 문서 생성 또는 저장 후 한글과 코드 블록 문자가 깨지지 않는지 확인한다.
- 인코딩 확인이 필요한 경우 `file -bi {파일명}` 명령으로 `charset=utf-8` 여부를 확인한다.

## 3. 구조

- `---` 구분선 사용 금지 - GitHub 및 Obsidian에서 헤더 하단 밑줄로 대체됨
- 헤더 레벨 건너뜀 금지 (H2 다음 H4 직접 사용 불가)
- H4 이하(`#####` 등) 사용 금지

헤더 번호 체계는 다음과 같다.

| 레벨 | 용도 | 번호 형식 |
|---|---|---|
| H1 `#` | 문서 제목 | 없음 |
| H2 `##` | 섹션 | `1.`, `2.` |
| H3 `###` | 서브섹션 | `1.1.`, `1.2.` |
| H4 `####` | 세부항목 | 없음 |

## 4. 파일명

- 단어 구분: 하이픈(`-`), 접두사 및 접미사 연결: 언더스코어(`_`)
- 날짜 형식: `YYMMDD-HHMMSS` (메타정보), `YYMMDD` (파일명)

```
lec-001_notes.md
260614-195052_session-handoff.md
```

## 5. Obsidian 최적화

### 5.1. 프론트매터

Obsidian에서 관리하는 문서에는 YAML 프론트매터를 작성한다.

```yaml
---
tags: [tag1, tag2]
created: YYYY-MM-DD
updated: YYYY-MM-DD
---
```

- `tags`: 문서 분류 태그 (소문자, 하이픈 구분)
- `created`: 문서 최초 작성일 - 최초 생성 시 기입, 이후 변경하지 않음
- `updated`: 문서 최종 수정일 - 내용 변경 시마다 에이전트가 갱신

### 5.2. 내부 링크

문서 간 연결은 Obsidian 내부 링크 형식을 사용한다.

```markdown
[[문서명]]
[[문서명|표시텍스트]]
```

링크 방향은 상위에서 하위로 향하는 단방향으로 제한한다. 하위 문서에서 상위 문서로 역방향 링크를 추가하지 않는다.

링크 추가 기준:

- 상위 문서가 하위 문서를 직접 참조하거나 목록으로 나열할 때 추가한다.
- 단순히 같은 폴더에 있다는 이유만으로 추가하지 않는다.
- 링크가 없어도 내용이 완결되는 문서에는 추가하지 않는다.

### 5.3. 태그

본문 내 태그는 `#태그명` 형식을 사용한다. 계층 태그는 슬래시로 구분한다.

```markdown
#딥러닝/CNN  #수치해석
```

### 5.4. 그래프 연결 원칙

- 관련 문서는 반드시 내부 링크로 연결하여 그래프 뷰에서 관계가 보이도록 한다.
- 고립 노드(링크 없는 문서)를 만들지 않는다.
